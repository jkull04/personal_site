# CODEX IMPLEMENTATION PROMPT

**Project:** Roman-themed personal site with restrained pixel dino guides
**Stack:** Plain HTML + CSS + minimal vanilla JS (no frameworks)
**Runtime:** Static, frontend-only

---

## 0. Core Concept

Build a 3-page personal site:

* `/projects.html`
* `/writings.html`
* `/contact.html`

Design philosophy:

* Roman = structure, typography, restraint.
* Dinos = minimal pixel sprites used sparingly for delight.
* Professional baseline first. Playful layer second.
* No gimmicks. No scroll hijacking. No excessive animation.

---

## 1. Non-Negotiables

### Accessibility

* WCAG 2.1 AA contrast (4.5:1 for body text minimum).
* Full keyboard navigability.
* Strong visible focus states.
* Respect `prefers-reduced-motion: reduce` (disable sprite animations and transitions).

### Motion Rules

* No scrolljacking.
* No infinite animation loops.
* All sprite animations ≤ 900ms.
* Animations trigger once per interaction.
* No autoplay beyond idle frame.

### Performance

* Max width images must define `width` and `height` attributes (avoid CLS).
* Lazy-load non-hero images.
* Sprite sheet loaded once and reused.
* No heavy libraries.
* Keep CSS lean and modular.
* No layout thrashing JS.

## 3. Global Layout Rules

* Max content width: 1040px
* Centered single-column layout
* 8px spacing system
* Section spacing: 64px vertical
* Card padding: 20px
* Border radius: 12px
* Hairline divider: 1px subtle border + tiny SVG endcap icon

### Top Navigation

Left: Name/Logo
Right: Projects · Writings · Contact
Active state: accent underline
Hover: subtle color shift

No sticky shrink effects. Keep stable.

---

## 4. Typography System

Use system stack or Inter.

Body:

* 16px base
* 1.6 line-height
* Neutral charcoal text

Headings:

* Same font family
* Small caps OR letter-spaced caps
* Museum placard feel
* Tight hierarchy (32 / 24 / 18 scale)

Metadata blocks:

* Small size
* Muted gray-green
* Format like:

```
Year · Tools · Outcome
```

---

## 5. Color Role System (CSS Variables)

Define semantic tokens:

```
--bg: bone/off-white
--text: charcoal
--muted: gray-green
--accent: dino green
--accent-2: darker green
--border: subtle divider gray
```

Accent used sparingly:

* Links
* Active nav
* Hover outlines
* Dino color

Avoid bright saturation outside accent.

## 7. Sprite System

Sprite sheet already exists: `/assets/sprites/dinos.png`

### Sprite Specs

* 32x32 per frame
* Integer scaling only
* Use `image-rendering: pixelated`
* Animate using `background-position` + `steps()`

### Required States Per Dino

* idle (1 frame)
* walk (4 frames)
* point or wave (2 frames)

### Interaction Mapping

Projects hero:

* One idle only

Project card hover:

* Dino performs 0.6–0.9s walk cycle once

Project detail first scroll into view:

* Dino points toward “Links” section once

Writings row hover:

* Dino bookmark pose (static after 1–2 frames)

Contact copy:

* “Sent” pose < 1s

### Reduced Motion

If `prefers-reduced-motion: reduce`:

* Disable sprite animations
* Show idle frame only
* Disable transitions

---

## 8. CSS Implementation Notes

* Use CSS variables for theme tokens.
* Use `steps(4)` animation for walk cycles.
* No keyframe loops beyond interaction duration.
* Sprite class structure:

```
.dino
.dino--walk
.dino--point
.dino--sent
```

Reset to idle after animation ends.
